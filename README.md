# 온라인/오프라인 게임 모음 배포본

## 포함된 기능
- 오프라인 플레이 유지
- 온라인 자동 매칭
- 서버 기준 턴 동기화
- 게임 중 채팅
- 에어하키 WebSocket 실시간 온라인 플레이
- 에어하키 온라인 패들 보간/입력 지연 완화
- 에어하키 서버 물리 계산 안정화

## 온라인 지원 게임
- 축고정 사목
- 오목
- 콰리도
- 땅따먹기
- 회전사목
- 별모양 점프 게임
- 브로커스
- 에어하키

## 프로젝트 구조
- `server.js`: 매칭, SSE 이벤트, 채팅, 정적 파일 서빙, 에어하키 WebSocket 실시간 동기화와 서버 기준 물리 계산
- `public/index.html`: 게임 UI, 기존 게임 로직, 에어하키 모바일/온라인 렌더링
- `public/client.js`: 온라인 매칭/채팅/턴 처리 클라이언트
- `package.json`: 배포용 실행 설정
- `render.yaml`: Render 배포용 설정 예시

## 로컬 실행 방법
```bash
npm start
```
그 다음 브라우저에서 `http://localhost:3000` 접속

## 공개 배포 방법
### Render
1. 이 폴더를 GitHub 저장소로 올립니다.
2. Render에서 새 Web Service를 만듭니다.
3. 저장소를 연결합니다.
4. Start Command를 `node server.js`로 설정합니다.
5. 배포가 끝나면 Render가 공개 URL을 발급합니다.

### Railway
1. 저장소를 GitHub에 올립니다.
2. Railway에서 New Project -> Deploy from GitHub Repo 선택
3. 시작 명령을 `node server.js`로 맞춥니다.
4. 배포 후 발급된 도메인으로 접속합니다.

## 주의
- 현재 서버는 메모리 기반 매칭 서버입니다. 서버가 재시작되면 대기열/방 정보는 초기화됩니다.
- 에어하키 온라인은 기존 턴제 게임의 SSE 방식과 별도로 WebSocket을 사용합니다.
- 장기 운영용으로는 방 만료 처리, 재접속 처리, 에러 로그, 보안 보완이 추가되면 좋습니다.

## v10.7.8.11 수정 사항

- 에어하키 온라인 서버 tick을 60fps 기준으로 변경
- 클라이언트 입력을 최신 값만 60fps로 전송하도록 변경해서 터치 입력 큐 적체 방지
- 서버 상태에 입력 ack/서버 tick 정보를 포함해 온라인 동기화 안정성 강화
- 에어하키 온라인 렌더링에서 WebSocket 수신 때마다 즉시 그리던 중복 렌더링 제거
- 점수/상태 DOM 갱신을 값이 바뀔 때만 하도록 줄여 모바일 버벅임 완화
- 서버 WebSocket heartbeat/ping 및 idle 연결 정리 추가


## v10.7.8.12 수정 사항

- 에어하키 온라인에서 오래된 서버 상태가 내 최신 패들 입력을 덮어써서 튀는 문제 보정
- 서버 물리는 60fps로 유지하되 상태 전송은 24fps로 조정하고 클라이언트 예측/보간으로 모바일 부담 완화
- 클라이언트 입력 ack가 확인되기 전에는 내 패들 로컬 위치를 우선하도록 보강
- WebSocket 클라이언트 프레임 masked 검증 추가로 깨진 프레임 차단
- 패들과 퍽 중심이 정확히 겹칠 때 충돌 계산이 무시되던 예외 처리 보강
- 비정상 좌표 입력이 들어와도 서버 물리 상태가 깨지지 않도록 좌표 범위 방어 코드 추가
